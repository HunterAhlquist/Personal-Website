Technical Details

The Smile64 uses a '64x64' resolution and refereshes every 50ms. Within that 50ms referesh, the computer works in two routines, 'console,' and 'microapp.' Microapps was a way to develop for the Smile64, only a few were made to showcase its features.

About the Project

This project was an experiment of limitations, for my 2020 Javascript class. This whole thing is inspired by Pico-8, a 'fantasy' game console built with limitations in mind, these limitations often lead to forcing one how to learn to optimize code, and memory. Not only that people have said that limitations enhance creativity. I've always been fascinted with old outdated development kits, such as the Commedore 64, and the many vintage console development kits released to studios in the past.

How it Works

Graphics

The 64x64 screen is made up via pixel object, defined in the pixel.js file, which also handles and defines many of the graphics related features such as fullscreen. The pixel object is basically just rgba color data, and those colors get placed into a 2D array named rasterizeData, every frame that data gets placed into a canvas renderer that emulates a pixel by creating a square representing said pixel.

Console/System

smilesystem.js handles many functions of the console, and input, and it updates under two modes; 'microapp' mode and console mode. This is so that the console doesnt take resources while a 'microapp' is running. Commands, are defined and handled in bin.js

microapps

"microapps," not "Microapps." No caps are to show how small they can be. You can even write your own! In order to register one you make, you have to load it into this html, and then register it in bin.js near the bottom in the exec function. Not the best way of handling stuff like that, but Windows still uses their registry so whatever. A microapp is loaded as an object and placed into the activeApp variable. All microapps initialize, update, and end. See the acid.js example here. The object definition acts as the initilization, this is also where you define your update, and other functions. update() will run automatically, along with any code inside after registering using:
app = { update : function(){update();}, end : function(){end();}, mem : [], input : new Object() };
This makes it so smilesystem.js can actually call the appropriate update() and end() functions, it also defines a memory variable which gets destoryed when the app closes. Additionally, and input variable is made, so that any keystrokes for the given frame can be passed to the microapp.
